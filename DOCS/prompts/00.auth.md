**Role:** Act as a Principal Django Architect and Security Engineer.

**Objective:** Create a production-grade REST API for Authentication using Django Rest Framework (DRF) and SimpleJWT.

**Task Requirements:**

1. **Model Layer (The Foundation):**
   - Implement a **Custom User Model** by extending `AbstractUser`.
   - Ensure it is ready for scalability (e.g., adding `is_verified` or `profile_picture` later).
2. **Security & JWT Configuration:**

   - Use `djangorestframework-simplejwt`.
   - **Token Rotation:** Enable `ROTATE_REFRESH_TOKENS` and `BLACKLIST_AFTER_ROTATION`.
   - **Lifetimes:** Configure access tokens for 15 minutes and refresh tokens for 7 days.

3. **API Endpoints (The 3 Paths):**

   - **Sign-Up:** Implement a `RegisterSerializer` that handles password hashing and basic validation.
   - **Login:** Use `TokenObtainPairView` but customize it to return user-specific data (ID, Username, Email) alongside the tokens.
   - **Refresh:** Implement `TokenRefreshView` and explain how the blacklist handles the old refresh token.

4. **Architectural Best Practices:**
   - Use **DRF Serializers** for data validation (mapping them to your NestJS DTO experience).
   - Use **Generic Views** or **ViewSets** for clean, readable code.
   - Separate the logic into a dedicated `users` or `identity` app.

**Output Format:**

- A clear directory structure.
- The code for `models.py`, `serializers.py`, and `views.py`.
- A brief explanation of the "Django Request Lifecycle" for this specific Auth flow.
